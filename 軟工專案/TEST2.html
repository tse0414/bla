示意說明：實際系統中，計費公式可為：

安全與權限

依角色限制功能可見性與資料存取。

角色與權限示意

客戶服務人員 倉儲人員 駕駛員 管理人員 系統管理員

上方右上角的「目前角色」下拉選單，目前僅用來「前端隱藏/顯示」不同區塊，方便 Demo。
實際系統中，後端 API 需再檢查呼叫者身分，例如在請求中一併送出 current_role 或 Token，若非管理員則拒絕敏感操作（對應需求 1.6 第 20, 21 點）。

  • 客戶：僅可看到「查詢包裹狀態」與自身貨件相關的資料。
  • 作業人員：可維護客戶資料、建立包裹、更新物流事件。
  • 管理員 / 系統管理員：可調整服務類型與定價、檢視所有帳單與統計資訊。
  • 即使前端隱藏按鈕，後端仍需驗證權限，避免「直接呼叫 API」造成未授權存取。
